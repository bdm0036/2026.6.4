# 网上书店微服务系统 — 部署指南

## 前提条件

| 软件 | 最低版本 | 检查命令 |
|------|---------|---------|
| Docker Desktop | 4.0+ | `docker version` |
| Node.js（可选） | 18+ | `node -v` |
| Git（可选） | 2.0+ | `git --version` |

## 第一步：配置 Docker 镜像加速

Docker Hub 在国内无法直连，必须配置镜像源。

1. 右键任务栏鲸鱼图标 → **Settings** → 左侧点 **Docker Engine**
2. 在 JSON 配置中加入 `registry-mirrors`，最终效果如下：

```json
{
  "builder": {
    "gc": {
      "defaultKeepStorage": "20GB",
      "enabled": true
    }
  },
  "experimental": false,
  "registry-mirrors": [
    "https://docker.m.daocloud.io"
  ]
}
```

3. 点击 **Apply & Restart**，等待 Docker 重启完成。
4. 打开终端（CMD 或 PowerShell），验证：

```bash
docker pull hello-world
```

看到 `Hello from Docker!` 说明配置成功。

## 第二步：启动项目

打开终端，进入项目的 docker 目录。

```bash
cd C:\Mycode\bookstore-system\docker
```

### 2.1 首次启动（需等待构建，约 5-10 分钟）

```bash
docker compose up -d
```

这个命令会自动完成以下所有工作：
- 拉取 MySQL、Redis、Consul 基础镜像
- 拉取 Maven 和 JRE 编译镜像
- 使用 Maven 编译 5 个后端微服务（第一次会下载 Spring Boot / MyBatis-Plus / JWT 等所有依赖）
- 编译 Vue 前端（npm install + npm run build）
- 启动全部 9 个容器
- 自动初始化 3 个数据库并插入测试数据

编译过程输出很多日志，看到所有容器 Started 即可。

### 2.2 之后每次启动（秒级）

以后开发需要重新构建时才加 `--build` 参数：

```bash
docker compose up -d                   # 不重新编译，直接启动
docker compose build --no-cache        # 忽略缓存，强制重新编译
docker compose build {服务名}           # 只重新编译某个服务
```

## 常用运维命令

```bash
# 查看所有容器状态
docker compose ps

# 查看某个服务的实时日志
docker compose logs -f gateway-service

# 重启某个出问题的服务
docker compose restart order-service

# 停止所有服务
docker compose down

# 完全停止并删除所有数据卷（数据库数据会丢失）
docker compose down -v

# 备份数据库
docker exec bookstore-mysql mysqldump -uroot -p123456 --all-databases > backup.sql

# 恢复数据库
docker exec -i bookstore-mysql mysql -uroot -p123456 < backup.sql

# 进入某个容器内部调试
docker exec -it bookstore-mysql mysql -uroot -p123456
```

## 故障排查

### 问题1：启动后某个服务不断重启

查看该服务的日志定位原因：

```bash
docker compose logs gateway-service | tail -30
```

常见原因：等待其他服务就绪（正常现象，会自动重试）、数据库连接失败、Consul 未注册成功。

### 问题2：端口冲突

如果本地 3306 端口已被占用，docker-compose.yml 中 MySQL 已改为映射 3307 端口。如果 80 或 8080 冲突，修改 docker-compose.yml 中对应服务的 ports 映射宿主机端口即可。

### 问题3：中文乱码

如果数据库中文显示乱码，执行以下步骤重建：

```bash
docker compose down -v      # 删除所有数据卷
docker compose up -d         # 重新启动（自动初始化）
```

### 问题4：Docker 引擎无法启动

```bash
# 完全关闭 Docker 和 WSL
powershell -Command "Stop-Process -Name 'Docker Desktop' -Force"
wsl --shutdown

# 删除锁文件
rm "$env:APPDATA\Local\Docker\backend.lock"
rm "$env:APPDATA\Local\Docker\frontend.lock"

# 重新启动 Docker Desktop
start "C:\Program Files\Docker\Docker\Docker Desktop.exe"
```
