## 一、你需要什么软件

| 软件 | 必须吗 | 用途 | 怎么装 |
|------|--------|------|--------|
| **Docker Desktop** | ✅ 必须 | 运行所有服务（数据库、后端、前端） | 见第二步 |
| Git | 推荐 | 拉取代码 | `winget install Git.Git` |
| 浏览器 | ✅ 必须 | 访问系统 | 你已经有 Chrome/Edge |
| Node.js | 可选 | 前端开发调试 | `winget install OpenJS.NodeJS` |

> **不需要装 Java、Maven、MySQL、Redis！** Docker 会搞定一切。

## 二、第一步：安装 Docker Desktop

### 2.1 下载安装

1. 打开浏览器，访问：**https://www.docker.com/products/docker-desktop/**
2. 点击 **Download for Windows**（约 600MB）
3. 下载完成后双击安装，一路点 **OK / Next / Install**
4. 安装完成后会提示**重启电脑**，必须重启

### 2.2 验证安装成功

重启后，按键盘 **Win + R**，输入 `cmd`，回车。在黑色窗口输入：

```bash
docker --version
```

看到类似 `Docker version 27.x.x` 说明安装成功。

> **注意：** Docker Desktop 启动需要 1-2 分钟。任务栏右下角出现鲸鱼图标，等它不再转圈。

## 三、第二步：配置 Docker 镜像加速

> **为什么做这步：** Docker 默认从海外拉取镜像，国内网络极慢甚至失败。配置镜像源后速度快 10 倍。**如果不做这步，后面的部署大概率失败。**

### 3.1 打开配置

1. 右键任务栏右下角的**鲸鱼图标** 🐳
2. 点击 **Settings**（设置）
3. 左侧菜单点击 **Docker Engine**

### 3.2 修改配置

你会看到一个 JSON 配置文本。在 `{ }` 大括号内**添加** `registry-mirrors` 这一项。

改完后的完整配置如下（直接复制粘贴替换）：

```json
{
  "builder": {
    "gc": {
      "defaultKeepStorage": "20GB",
      "enabled": true
    }
  },
  "experimental": false,
  "registry-mirrors": [
    "https://docker.m.daocloud.io"
  ]
}
```

### 3.3 应用配置

点击右下角的 **Apply & Restart** 按钮。Docker 会自动重启（约 30 秒）。

### 3.4 验证配置

打开终端（Win + R → 输入 `cmd`），输入：

```bash
docker pull hello-world
```

看到 `Hello from Docker!` 就说明配置成功了。

## 四、第三步：启动整个系统

> **前提：** 项目代码已经在你的电脑上。如果还没有，先克隆：
> ```bash
> git clone <你的GitLab地址>
> cd bookstore-system
> ```
> 或者直接从 U 盘/网盘复制 `bookstore-system` 文件夹到桌面。

### 4.1 打开终端

按 **Win + R**，输入 `cmd`，回车。

### 4.2 进入项目目录

```bash
cd C:\你的路径\bookstore-system\docker
```

> **提示：** 如果你把项目放在桌面，路径大概是：
> ```bash
> cd C:\Users\你的用户名\Desktop\bookstore-system\docker
> ```
> 在文件资源管理器中打开 `docker` 文件夹，点击地址栏，复制路径，然后在终端里输入 `cd` + 空格 + 粘贴路径。

### 4.3 一键启动（首次约 10-15 分钟，之后秒起）

```bash
docker compose up -d
```

**这条命令做了什么？**
- 自动拉取 MySQL、Redis、Consul 镜像
- 自动编译 5 个后端微服务
- 自动编译 Vue 前端
- 自动创建 3 个数据库并建表
- 启动全部 9 个容器

你会看到屏幕上不断滚动的日志。**耐心等待**，直到出现全部 `Started`。

### 4.4 查看是否全部启动成功

```bash
docker compose ps
```

正常情况下，你会看到 **9 行**，全部显示 `Up`：

```
NAME                       STATUS
bookstore-consul           Up
bookstore-mysql            Up (healthy)
bookstore-redis            Up (healthy)
bookstore-auth-service     Up
bookstore-user-service     Up
bookstore-product-service  Up
bookstore-order-service    Up
bookstore-gateway          Up
bookstore-frontend         Up
```

> **如果没有 9 行或有显示 `Exited` 的**，跳到[第八章：常见问题排查](#八常见问题排查)。

打开浏览器，输入：**http://localhost**

### 5.3 测试登录

在 http://localhost 页面：

2. 输入 `admin` / `123456`
3. 登录成功后导航栏出现蓝色**「管理后台」按钮**

## 七、第六步：关闭和重启

### 关闭系统（保留数据）

```bash
cd C:\你的路径\bookstore-system\docker
docker compose down
```

下次启动：

```bash
docker compose up -d
```

**第二次启动只需要 10-30 秒**，因为镜像已经构建好了。

### 完全删除（包括数据库数据）

```bash
docker compose down -v
```

## 八、常见问题排查

### 问题 1：`docker compose up -d` 卡住或报错

**现象：** 终端长时间没反应，或出现 `error pulling image`、`connection refused`

**原因：** 镜像加速没配好。

**解决：**
1. 检查第三步的镜像加速配置是否正确
2. 重启 Docker Desktop
3. 再试 `docker pull hello-world` 确认加速可用
4. 然后重新 `docker compose up -d`

---

### 问题 2：端口冲突

**现象：** `port is already allocated` 或 `bind: address already in use`

**原因：** 你的电脑上 80、3306、6379、8080、8500 等端口被其他程序占用了。

**解决：**
```bash
# 查看谁在占用端口（以 80 为例）
netstat -ano | findstr :80

# 记下 PID（最后一列数字），去任务管理器结束对应进程
# 常见冲突：XAMPP、MySQL、IIS、迅雷
```

---

### 问题 3：Consul 里没有 5 个绿色对勾

**现象：** http://localhost:8500 打开后 Services 为空，或有红色 X

**原因：** 服务还在启动中，或某个服务崩溃了。

**解决：**
```bash
# 等 30 秒再看
# 如果还是没有，重启有问题的服务
docker restart bookstore-product-service
docker restart bookstore-order-service
```

---

### 问题 4：登录失败，报 500 错误

**原因：** 数据库表还没建好。

**解决：**
```bash
# 等 30 秒，让 init.sql 执行完
# 如果还不行，手动执行
docker exec -i bookstore-mysql mysql -uroot -p123456 < ../sql/init.sql
```

---

### 问题 5：前端页面空白，控制台报 CORS 错误

**原因：** 前端访问的 API 地址不对。

**解决：** 确认浏览器访问的是 `http://localhost`（不是 `localhost:5173`）。Docker 部署后的前端在 80 端口。

---

### 问题 6：Docker Desktop 打不开

**解决：**
```bash
# 以管理员身份打开 PowerShell，执行：
wsl --shutdown

# 然后重新启动 Docker Desktop
```

---

### 问题 7：改了前端代码但不生效

**原因：** Docker 运行的是构建后的版本，改源码不会自动更新。

**解决：**

**方式一（推荐，30 秒）：**
```bash
cd frontend
npm run build
docker cp dist/. bookstore-frontend:/usr/share/nginx/html/
```

**方式二（完整重建，5 分钟）：**
```bash
cd docker
docker compose build frontend
docker compose up -d frontend
```

---

### 问题 8：改了后端代码但不生效

**解决：**
```bash
cd docker
docker compose build <服务名>    # 比如：docker compose build user-service
docker compose up -d <服务名>
```

## 九、附录：手动开发模式（可选）

> 如果你需要频繁改前端代码，每次 `docker cp` 太慢，可以用开发模式。**但需要安装 Node.js。**

### 9.1 启动后端（Docker）

```bash
cd docker
docker compose up -d consul mysql redis auth-service user-service product-service order-service gateway-service
```

### 9.2 启动前端（本地开发服务器）

```bash
cd frontend
npm install          # 第一次才需要
npm run dev          # 启动开发服务器
```

访问 **http://localhost:5173**（自动热更新，改代码立刻看到效果）。

## 快速命令速查表

| 想做什么 | 命令 |
|---------|------|
| 一键启动全部 | `cd docker && docker compose up -d` |
| 查看运行状态 | `docker compose ps` |
| 停止全部 | `docker compose down` |
| 停止并清空数据 | `docker compose down -v` |
| 查看某服务日志 | `docker logs -f bookstore-gateway` |
| 重启某服务 | `docker restart bookstore-frontend` |
| 重建某服务 | `docker compose build frontend && docker compose up -d frontend` |
| 进入容器内部 | `docker exec -it bookstore-mysql mysql -uroot -p123456` |
| 备份数据库 | `docker exec bookstore-mysql mysqldump -uroot -p123456 --all-databases > backup.sql` |
| 恢复数据库 | `docker exec -i bookstore-mysql mysql -uroot -p123456 < backup.sql` |
| 修改前端后更新 | `cd frontend && npm run build && docker cp dist/. bookstore-frontend:/usr/share/nginx/html/` |
